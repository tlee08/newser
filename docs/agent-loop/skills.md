# Skills Map

Use these skills and practices when continuing this repo.

## Always Start

- `superpowers:using-superpowers`: check whether a skill applies before acting.
- Read `AGENTS.md` and `docs/agent-loop/context.md`.

## Product or Feature Changes

- `superpowers:brainstorming`: use before changing behavior, UX, or scope.
- `superpowers:writing-plans`: use for multi-step implementation.
- `superpowers:test-driven-development`: use for bug fixes and feature logic where tests can define behavior.

## Frontend Work

- `frontend-design`: use when modifying the visual direction, layout, interaction model, or copy tone.
- `context7-mcp`: fetch current docs for React, Mantine, Vite, or NewsAPI-adjacent libraries before changing library-specific code.

## Debugging

- `superpowers:systematic-debugging`: use before fixing broken builds, runtime errors, failed API calls, or unexpected UI behavior.
- `superpowers:verification-before-completion`: use before saying the work is done.

## Repo Loop

1. Identify the next narrow task.
2. Confirm relevant skill instructions.
3. Write or update a focused test when practical.
4. Implement the smallest coherent change.
5. Run verification commands.
6. Update docs when setup, behavior, or architecture changes.
